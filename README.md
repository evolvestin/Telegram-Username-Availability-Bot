## Описание
Данный проект представлял собой Telegram-бота для проверки доступности публичных ссылок (юзернеймов) в режиме Inline. Бот выполнял предварительную валидацию синтаксиса, после чего отправлял запросы к веб-версии Telegram для определения статуса занятости адреса.

## Логика валидации
Процесс проверки имени пользователя осуществлялся в два этапа:

1.  **Синтаксическая и лексическая проверка** (Regex).
2.  **Проверка доступности** (HTTP-запрос).

### Критерии валидации и ответы системы

Ниже приведен список сценариев валидации и соответствующих ответов бота пользователю:

| Сценарий | Ответ бота | Причина |
| :--- | :--- | :--- |
| **Недопустимые символы** | `Only 0-9, a-z, and underscores allowed.` | Использованы символы, отличные от латиницы, цифр или нижнего подчеркивания. |
| **Синтаксическая ошибка** | `This link is invalid` | Юзернейм начинается с цифры или содержит два нижних подчеркивания подряд (`__`). |
| **Недостаточная длина** | `Sorry, this link is too short.` | Длина запроса менее 5 символов. |
| **Адрес занят** | `Sorry, this link is already occupied.` | Страница существует, адрес зарегистрирован другим пользователем. |
| **Адрес свободен** | `This link is available.` | Страница не найдена или доступна для регистрации (согласно старой логике). |

## Технические особенности реализации

*   **Асинхронность:** Бот написан с использованием библиотеки `aiogram` и модуля `asyncio`.
*   **Многопоточность:** Для ускорения проверки группы адресов (например, с постфиксами `_bot`) использовался `concurrent.futures.ThreadPoolExecutor`.
*   **Парсинг:** Анализ доступности осуществлялся посредством библиотеки `BeautifulSoup4` путем проверки наличия определенных мета-тегов и CSS-классов на странице `t.me/username`.
